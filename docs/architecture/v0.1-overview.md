# EduMed v0.1 - Architecture Overview

## System Overview

EduMed is a monorepo educational management system built with Node.js backend and React frontend, designed for schools to manage journals, timetables, documents, and statistics.

## High-Level Architecture

```
┌─────────────┐
│   Web App   │ (React/Next.js)
│  (Port 3000)│
└──────┬──────┘
       │ HTTP/REST
       ▼
┌─────────────┐
│   API       │ (Node.js + TypeScript)
│  (Port 4000)│
└──────┬──────┘
       │ SQL
       ▼
┌─────────────┐
│  PostgreSQL │
│  (Port 5432)│
└─────────────┘
```

## Core Modules (v0.1)

### 1. Authentication & Authorization
- JWT-based authentication
- Role-based access control (RBAC)
- Roles: Admin, Teacher, Student, Parent
- Permission system for fine-grained access

### 2. User Management
- User CRUD operations
- Role assignment
- Profile management

### 3. School Structure
- School information
- Class management
- Subject management
- Course management (class + subject + teacher)

### 4. Journal Module
- Grade tracking
- Homework assignments
- Attendance recording
- Comments and notes

### 5. Timetable Module
- Schedule creation and management
- Lesson planning
- Room assignment
- Conflict detection

### 6. Document Management
- File upload and storage
- Document categorization
- Access control per document
- Support for various file types

### 7. Statistics Module
- Student performance analytics
- Class statistics
- Subject-wise reports
- Attendance reports

### 8. Personal Dashboards
- Teacher dashboard: classes, schedule, grading
- Student dashboard: grades, homework, schedule
- Parent dashboard: child's progress
- Admin dashboard: system overview

## Technology Decisions

### Backend
- **Node.js + TypeScript**: Type safety, modern JS features
- **Express/Fastify**: Lightweight, flexible routing
- **PostgreSQL**: Relational data, complex queries, ACID compliance
- **Prisma (optional)**: Type-safe ORM, migration management

### Frontend
- **React/Next.js**: Component-based, SSR capability
- **TypeScript**: Type safety across stack
- **Modern design system**: Reusable components

### Infrastructure
- **Docker**: Consistent development environment
- **Docker Compose**: Multi-container orchestration
- **npm workspaces**: Monorepo management

## Data Flow

1. **User Request** → Web frontend
2. **API Call** → Backend validates JWT
3. **Authorization** → Check user permissions
4. **Business Logic** → Service layer processes request
5. **Data Access** → Repository layer queries database
6. **Response** → JSON returned to frontend
7. **UI Update** → React components re-render

## Security Considerations

- Password hashing (bcrypt)
- JWT token rotation
- Input validation and sanitization
- SQL injection prevention (parameterized queries)
- Rate limiting
- CORS configuration
- File upload validation

## Scalability Considerations

- Stateless API design
- Database indexing on frequently queried fields
- Pagination for large datasets
- File storage externalization (S3/Firebase)
- Caching strategy (Redis - future)
